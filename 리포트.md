## 1. 보안 강화 (Priority: P0 — 즉시)

### 1-1. 관리자 비밀번호 해싱

- **현재**: `backend/app/api/auth.py` — `secrets.compare_digest(password, settings.admin_password)` 평문 비교
- **문제**: 비밀번호가 메모리/환경변수에 평문 저장, 로그 노출 위험
- **계획**:
  - `passlib[bcrypt]` 또는 `bcrypt` 의존성 추가
  - 초기 기동 시 환경변수 비밀번호를 bcrypt 해시로 변환하여 메모리 보관
  - `verify_password()`를 `bcrypt.checkpw()` 기반으로 교체
  - `.env.example`에서 기본 비밀번호 `admin123` 제거

### 1-2. 세션 관리 개선

- **현재**: `auth.py` — 인메모리 딕셔너리 토큰 저장, 서버 재시작 시 전체 소실
- **계획**:
  - JWT(JSON Web Token) 기반으로 전환 — `python-jose` 또는 `PyJWT` 사용
  - Access Token (1h) + Refresh Token (7d) 이중 구조
  - 토큰에 만료시간 내장 → 서버 상태 불필요

### 1-3. Nginx 보안 헤더

- **현재**: `nginx/nginx.conf` — 보안 헤더 없음
- **계획**: 아래 헤더 추가
  ```nginx
  add_header X-Frame-Options "SAMEORIGIN" always;
  add_header X-Content-Type-Options "nosniff" always;
  add_header X-XSS-Protection "1; mode=block" always;
  add_header Referrer-Policy "strict-origin-when-cross-origin" always;
  add_header Content-Security-Policy "default-src 'self'; script-src 'self' 'unsafe-eval'; style-src 'self' 'unsafe-inline';" always;
  ```

### 1-4. CORS 범위 축소

- **현재**: `backend/app/main.py` — `allow_methods=["*"]`, `allow_headers=["*"]`
- **계획**: 실제 사용하는 메서드/헤더만 명시
  ```python
  allow_methods=["GET", "POST", "DELETE"],
  allow_headers=["Authorization", "Content-Type"],
  ```

### 1-5. API 엔드포인트별 Rate Limiting

- **현재**: 전역 IP 기반 120req/min만 존재
- **계획**:
  - `/api/admin/login` → 5req/min (브루트포스 방지)
  - `/api/graph` → 30req/min (DB 부하 방지)
  - `/api/search` → 60req/min
  - Rate Limit 응답 헤더 포함 (`X-RateLimit-Remaining`, `Retry-After`)

## 2. 프론트엔드 아키텍처 개선 (Priority: P1)

### 2-1. GraphView3DLarge.jsx 분리 (1,553줄 → 모듈화)

현재 하나의 파일에 셰이더, 카메라 제어, 라벨 관리, 선택 하이라이팅, 블룸 효과가 모두 집약되어 유지보수가 어려움.

**분리 계획:**

```
components/graph/
├── GraphView3DLarge.jsx      # 메인 컨테이너 (~300줄)
├── shaders/
│   └── celestialBody.js      # 노드 셰이더 머티리얼 정의
├── systems/
│   ├── bloomManager.js       # UnrealBloom 초기화 + 동적 조절
│   ├── labelPool.js          # 스프라이트 라벨 풀 관리
│   ├── flyControls.js        # 키보드 비행 컨트롤
│   ├── selectionHighlight.js # 3-hop BFS + 색상 그래디언트
│   └── cameraAnimation.js    # animateCamera 유틸
├── hooks/
│   ├── useGraphSetup.js      # Effect 2: Three.js 씬 초기화
│   ├── useSelectionVisual.js # Effect 3: 선택 비주얼
│   └── useStyleSync.js       # Effect 4: 스타일 실시간 반영
└── constants.js              # NODE_COLORS, MAX_LABELS 등
```

### 2-2. 에러 핸들링 UI

- **현재**: `catch (_) {}` — fetch 에러를 조용히 무시 (GraphPage.jsx 여러 곳)
- **계획**:
  - 경량 토스트 알림 컴포넌트 구현 (외부 라이브러리 없이)
  - fetch 실패 시 사용자에게 "네트워크 오류" / "서버 응답 없음" 표시
  - 크롤러 상태 폴링 실패 시 "연결 끊김" 배지 표시

### 2-3. 성능 최적화

**레이블 텍스처 아틀라스:**
- **현재**: 노드마다 개별 Canvas → Texture 생성, 500개 캐시 후 eviction
- **계획**: 공통 폰트/크기의 글리프를 하나의 아틀라스 텍스처에 통합
- **효과**: GPU 텍스처 바인드 횟수 대폭 감소, VRAM 절약

**인접맵 캐싱:**
- **현재**: `adjacencyMap`은 useMemo로 캐싱되지만 `highlightSet`이 매 선택마다 BFS 재계산
- **계획**: 이미 useMemo 적용됨 — 추가 개선 시 레이아웃 확정 후 프리빌트 인접 인덱스 저장

### 2-4. Nginx 정적 에셋 최적화

```nginx
# gzip 압축 (전송 크기 60-70% 감소)
gzip on;
gzip_types text/plain text/css application/json application/javascript text/xml;
gzip_min_length 1000;

# 버전된 에셋 캐싱 (Vite 해시 파일명)
location /assets/ {
    expires 1y;
    add_header Cache-Control "public, immutable";
}
```

## 3. 백엔드 아키텍처 개선 (Priority: P1)

### 3-1. 데이터베이스 마이그레이션 체계화

- **현재**: `database.py` — 수동 `ALTER TABLE` + 테이블 존재 체크
- **문제**: 스키마 변경 이력 추적 불가, 복잡한 마이그레이션 위험
- **계획**:
  - Alembic 도입
  - `backend/alembic/` 디렉토리 구조 생성
  - 기존 수동 마이그레이션 코드를 초기 마이그레이션으로 변환
  - `alembic upgrade head`를 Docker CMD에 통합

### 3-2. 그래프 빌더 쿼리 최적화

- **현재**: `builder.py` — 대규모 그래프에서 coworker 링크 계산이 O(n²)
- **계획**:
  - 공유 레포 수 기반 coworker를 DB 단에서 사전 집계 (Materialized View 또는 캐시 테이블)
  - 그래프 빌드 시 집계 테이블에서 직접 조회
  - 크롤링 완료 시 비동기로 집계 테이블 갱신

### 3-3. Uvicorn 워커 설정

- **현재**: 단일 워커 — 동시 요청 처리 제한
- **계획**:
  ```dockerfile
  CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000", "--workers", "4"]
  ```
  또는 Gunicorn + Uvicorn Worker 조합:
  ```dockerfile
  CMD ["gunicorn", "app.main:app", "-w", "4", "-k", "uvicorn.workers.UvicornWorker", "--bind", "0.0.0.0:8000"]
  ```

### 3-4. 검색 기능 강화

- **현재**: `routes.py` — `ilike` 패턴 매칭, 결과 20개 고정, 페이지네이션 없음
- **계획**:
  - PostgreSQL `pg_trgm` 확장 활용한 트라이그램 인덱스
  - offset/limit 기반 페이지네이션 추가
  - 검색 결과에 관련도(relevance) 점수 포함

## 4. 시각화 효과 고도화 (Priority: P2)

### 4-1. 선택적 블룸 (Selective Bloom)

- **현재**: 전체 씬에 블룸 적용 → 비선택 노드도 약간의 블룸 발생
- **계획**: 듀얼 렌더 패스 방식
  1. **블룸 대상 레이어**: 선택된 노드 + 3-hop 이웃만 별도 레이어에 배치
  2. **일반 레이어**: 나머지 노드는 블룸 없이 렌더
  3. 두 패스를 합성하여 최종 출력
- **효과**: 선택 노드만 정확하게 빛나고, 배경은 완전히 깔끔

### 4-2. 엣지 애니메이션

- **현재**: 엣지는 정적 LineSegments, 색상만 변경
- **계획**:
  - 선택 시 엣지를 따라 흐르는 파티클/대시 애니메이션 (셰이더 기반)
  - `dashSize` + `gapSize` uniform을 시간에 따라 오프셋
  - Hop 거리에 따라 파티클 속도/밀도 차등

### 4-3. 노드 크기 펄스 효과

- **현재**: 선택 노드에 링 효과만 존재
- **계획**:
  - 선택된 노드의 스케일을 사인파로 미세 펄스 (±5%)
  - Hop 1 이웃도 약한 펄스 적용 (±2%)
  - `animate()` 루프에서 `instanceMatrix` 스케일 업데이트

### 4-4. 깊이감 향상

- **현재**: FogExp2 단순 안개, 15k 노드 미만에서만 적용
- **계획**:
  - SSAO(Screen Space Ambient Occlusion) 패스 추가 — 밀집 영역에 자연스러운 그림자
  - 대규모 그래프에서도 적응적 fog density 적용 (현재는 비활성)
  - 선택 모드에서 fog density를 높여 비선택 영역을 더 흐리게

### 5-2. 프론트엔드 테스트

- **프레임워크**: Vitest + @testing-library/react
- **우선 테스트 대상**:
  ```
  __tests__/
  ├── GraphPage.test.jsx       # 페이지 통합 테스트
  ├── Sidebar.test.jsx         # 필터 상호작용
  ```

### 5-3. CI/CD 파이프라인

```yaml
# .github/workflows/ci.yml
name: CI
on: [push, pull_request]
jobs:
  backend:
    - pytest --cov --cov-report=xml
    - ruff check (린팅)
  frontend:
    - npm run build (빌드 검증)
    - vitest run --coverage
  docker:
    - docker compose build (이미지 빌드 검증)
```

### 5-4. 코드 품질 도구

- **백엔드**: `ruff` (Python linter + formatter, Black 대체)
- **프론트엔드**: `eslint` + `prettier`
- **Pre-commit hook**: `pre-commit` 프레임워크로 커밋 전 자동 검사

## 6. 운영 & 모니터링 (Priority: P2)

### 6-1. 구조화된 로깅

- **현재**: 표준 Python `logging`, 포맷 미설정
- **계획**:
  - `structlog` 도입 → JSON 포맷 로그
  - 요청 ID(correlation ID) 전파
  - Docker 로그 드라이버로 수집 경로 설정

### 6-2. 헬스체크 강화

- **현재**: DB 연결만 확인하는 `/api/health`
- **계획**:
  ```json
  {
    "status": "healthy",
    "database": "connected",
    "crawler": { "running": true, "workers": 3, "queue_depth": 42 },
    "cache": { "entries": 5, "hit_rate": 0.87 },
    "uptime_seconds": 86400
  }
  ```
  - Docker health check에 이 엔드포인트 활용

### 6-3. 데이터베이스 백업

- **계획**:
  - `pg_dump` 기반 일일 자동 백업 (cron 또는 별도 컨테이너)
  - 최근 7일 보관, 주간 백업은 30일 보관
  - docker-compose에 백업 볼륨 마운트

### 6-4. Docker 리소스 제한

```yaml
services:

    deploy:
      resources:
        limits:
          memory: 1G
```
